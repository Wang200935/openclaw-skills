# Delivery Discipline

Use this skill when a task risks turning into endless analysis without completion.

## Core lessons

1. Once the root cause is known, patch immediately before exploring side branches.
2. Do not treat architecture notes as delivered functionality.
3. Prefer one working vertical slice over many clever partials.
4. Verify every claimed milestone before reporting it.
5. Do not tell the user something is done because one subcomponent exists.

## Execution order

- identify main deliverable
- isolate blockers
- patch blocker
- test
- only then broaden into references, polish, and extra skills

## Required reporting discipline

When asked for progress, distinguish clearly:
- implemented
- integrated
- verified once
- stable over time
- fully ready for handoff
