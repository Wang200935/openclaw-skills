# NLP / 公開文字資料題型操作手冊

這份文件把 `nlp-topic-templates.md` 擴充成可直接落地的 v2 操作版。

目標不是把高中小論文做成研究所論文，而是把題目收斂到「拿得到資料、做得出方法、講得清結果、寫得出限制」的程度。

## 為什麼這類題目特別適合高中資訊小論文

根據本地 404 篇資訊類小論文語料與 chunk reviews，可重複看到幾個現象：

- 公開文字資料比自建大型影像資料更容易取得與重現
- 題目若同時有資料、分類或分析方法、量化結果，通常比純議題整理更有競爭力
- 問卷型社會議題若沒有客觀資料支撐，容易淪為「大家覺得如何」
- NLP / 公開文字題只要範圍縮小，往往能做出明確圖表、錯誤分析與比較結果
- 這類題目很適合結合 Python、爬蟲、資料清理、基礎機器學習、視覺化與小型網站展示

## 什麼叫做「小論文安全範圍」

最常見的失敗不是太弱，而是做太大。

### 建議安全範圍
- 樣本量：先求 500-5000 筆可處理文字，再看能力擴充
- 類別數：先做 2-5 類，不要一開始就 20 類
- 模型數：先做 1 個 baseline + 1 個較強方法即可
- 任務數：一篇只解決 1 個核心問題，不要同時做情緒分析、主題分析、預測、推薦、聊天機器人
- 平台數：先鎖定 1 個資料來源，不要同時抓 Threads + Instagram + YouTube + Dcard

### 不建議的危險範圍
- 想證明「整個世代」或「全台灣高中生」的想法
- 想做超大型自訓練 LLM
- 想處理太多語言、太多平台、太多主題
- 沒有標註規則就直接訓練分類器
- 只有模型準確率，沒有資料來源、沒有錯誤分析、沒有限制說明

## 可用資料來源選單

優先選擇：穩定、可記錄、能重抓、來源可交代的資料。

### A. 較穩定的公開文字來源
- Google Play / App Store 評論
- 新聞標題與新聞中繼資料
- 公開論壇文章或留言（在合法且合比例前提下）
- YouTube 公開留言
- 政府開放資料中的文字欄位
- 官方公開聲明、公告、新聞稿

### B. 可考慮但要更小心的來源
- Threads 公開貼文或互動資料
- Dcard / PTT 公開文章
- 公開粉專留言

### C. 選來源時要記錄的東西
- 抓取日期
- 關鍵字
- 篩選條件
- 排除條件
- 實際保留筆數
- 每筆資料的分析單位是什麼（貼文、留言、句子、評論）

## 一條完整研究流程

## 1. 先把題目改寫成可驗證問題

不要寫：
- 探討社群媒體對青少年的影響
- 分析 AI 假訊息的危害

改寫成：
- 能否利用公開健康新聞文本，建立繁中假訊息二元分類模型？
- 能否從匿名留言中辨識高風險霸凌語氣並建立預警分數？
- App 評論中最常見的負評類型是哪些，且哪種分類方法效果較好？

### 好研究問題的格式
- 對象：哪一種文字資料
- 任務：分類 / 情緒分析 / 主題分析 / 預測
- 比較：哪兩種方法或哪幾類標籤
- 證據：用什麼指標證明

## 2. 定義分析單位

常見單位：
- 一則留言
- 一篇貼文
- 一則評論
- 一個新聞標題
- 一則匿名投稿

如果單位不清楚，後面標註、切分、統計都會亂掉。

## 3. 設計標註規則

任何分類任務都要先回答：
- 類別定義是什麼？
- 灰色地帶怎麼判？
- 一筆資料能不能多標？
- 標不出來的資料怎麼處理？

### 最低要求
至少做出一份標註說明表：
- 類別名稱
- 類別定義
- 正例 2-3 句
- 反例 2-3 句
- 邊界案例處理方式

## 4. 清理資料

常做的事：
- 去重複
- 去空白或極短文本
- 統一編碼
- 保留 / 移除 emoji 的原則要先決定
- 移除明顯無意義內容（只有網址、只有標點）
- 記錄清理前後筆數

## 5. 建 baseline

小論文很需要 baseline，因為沒有 baseline 就很難說你做得好。

### 常見 baseline
- 關鍵字規則法
- TF-IDF + Logistic Regression
- TF-IDF + Naive Bayes
- 簡單詞頻 / 情緒詞典法

### 可再往上一層的方法
- XGBoost
- 中文 BERT / ALBERT 類模型
- 句向量 + 傳統分類器

### 安全原則
如果資料不大，先把 baseline 做穩，比硬上深度模型更重要。

## 6. 評估

至少選 2-3 個指標，不要只報一個 accuracy。

### 分類任務常用
- Accuracy
- Precision
- Recall
- F1-score
- Confusion matrix

### 排序 / 預警任務可補充
- Top-k precision
- Recall at threshold
- ROC-AUC（若有能力）

### 分析型任務常用
- 類別分布圖
- 時間趨勢圖
- 關鍵詞比較
- 主題比例圖

## 7. 做錯誤分析

這是很多作品缺的地方，也是拉開差距的地方。

至少回答：
- 模型最常把哪兩類搞混？
- 是因為反諷、縮寫、語氣詞、上下文不足，還是標註本身就模糊？
- 哪些資料其實不適合只看單句判斷？

## 8. 寫限制

不要怕寫限制。誠實的限制會讓作品更像研究。

常見限制：
- 來源只來自單一平台
- 資料時間區間有限
- 標註由少數研究者完成，主觀性仍存在
- 公開資料不等於整體族群
- 模型只處理文本，未納入圖片、脈絡、回覆串

## 六種高可行題型模板

## 題型 1：假訊息 / 錯誤資訊分類

### 適合原因
- corpus 中有明顯強例：繁中假訊息偵測、醫療假訊息、詐騙簡訊辨識
- 容易形成「資料 -> 標註 -> 模型 -> 指標 -> 網頁展示」完整鏈

### 可用資料
- 事實查核機構已查核文本
- 健康 / 醫療錯誤資訊案例
- 詐騙簡訊文本
- 新聞標題與已知真假標籤

### 分析單位
- 一則訊息
- 一段新聞文字
- 一個標題

### 標籤範例
- 真 / 假
- 詐騙 / 非詐騙
- 高風險 / 低風險

### baseline
- TF-IDF + Logistic Regression
- Naive Bayes

### 較強方法
- 中文 BERT / ALBERT
- XGBoost 搭配文字特徵

### 評估指標
- Precision / Recall / F1
- Confusion matrix

### 可做圖表
- 各模型 F1 比較圖
- 常見誤判詞彙
- 假訊息常見語氣特徵

### 小論文寫作角度
- 為什麼繁中資料值得獨立研究
- 醫療或校園場景的實務價值
- 傳統模型與 BERT 類模型的差異

### 常見限制
- 標註來源可能偏向特定主題
- 假訊息定義依賴查核標準
- 標題與全文資訊量不同

## 題型 2：網路霸凌 / offensive language 偵測

### corpus 提示
- backup corpus 已有 offensive language 類高價值例子
- chunk_08 也出現匿名留言、霸凌預警這種方向

### 可用資料
- 匿名留言板
- 公開留言區
- 校園意見箱類文本
- offensive comment corpus

### 標籤範例
- 霸凌 / 非霸凌
- 攻擊 / 中性 / 支持
- 高風險語氣 / 一般負面 / 無風險

### 難點
- 反諷、玩笑、熟人互動很難判
- 單句常缺脈絡

### baseline
- 關鍵字規則法
- TF-IDF + Logistic Regression

### 較強方法
- BERT 類分類器
- 情緒與攻擊詞特徵混合模型

### 評估補充
- 特別看高風險類的 recall
- 列出誤判案例

### 可做圖表
- 風險類別比例圖
- 常見攻擊詞 / 語氣詞統計
- 模型混淆矩陣

### 小論文寫作角度
- 校園預警不是取代輔導，而是輔助篩查
- 匿名留言的語氣比單純正負向更重要

## 題型 3：App 評論探勘與抱怨分類

### 為什麼很適合
- 資料容易取得
- 商業與生活應用感強
- 很容易做出主題分群與負評分類

### 可用資料
- Google Play 評論
- App Store 評論

### 常見研究問題
- 哪些負評類型最常見？
- 更新前後負評主題有沒有變化？
- 傳統分類器與深度模型哪個較適合？

### 標籤範例
- 閃退 / 介面 / 廣告 / 付款 / 速度 / 帳號 / 其他
- 正評 / 中評 / 負評

### baseline
- 詞頻與關鍵字分類
- TF-IDF + Logistic Regression

### 較強方法
- BERT 類模型
- K-means / topic modeling 做輔助分群

### 視覺化
- 抱怨類型長條圖
- 星等與評論主題交叉表
- 版本更新前後差異圖

### 常見限制
- 評論很短
- 星等不一定等於真實情緒
- 可能有灌水或重複留言

## 題型 4：社群平台互動與語意分析

### corpus 提示
- Threads / Instagram / VTuber / 平台互動偏好在語料中很多
- 但純問卷版本常偏弱；若加入公開文字或互動資料，會更紮實

### 可用資料
- 公開 Threads 貼文
- YouTube 留言
- 平台公開貼文中繼資料

### 研究問題範例
- 文案長度與互動量是否有關？
- 正負向語氣與留言回應數是否有關？
- 不同主題貼文的互動型態有何差異？

### 適合方法
- 描述統計
- 簡單情緒分析
- 主題分類
- 迴歸或分組比較（若能力足夠）

### 強化方式
不要只做「大家比較喜歡哪個平台」，改成：
- 直接抓公開貼文資料
- 分析字數、時段、語氣與互動量
- 再用少量問卷補充動機解釋

## 題型 5：學生 AI 使用論述 / 焦慮 / 教育討論探勘

### 為什麼值得做
- 和校園情境貼近
- 容易收集公開文字
- 可從社會議題轉成有資料的文字分析題

### 可用資料
- 公開論壇上的學習、考試、AI 工具討論串
- 新聞留言
- 校園公開討論文本

### 可做任務
- 主題分析：常見 concern 是什麼
- 立場分類：支持 / 保留 / 反對
- 情緒分析：焦慮、期待、中性

### 小心點
不要直接聲稱「高中生整體想法」，應寫成「在所蒐集的公開討論文本中，常見論述類型為……」。

## 題型 6：新聞標題情緒 / 文字特徵與下游結果

### 例子
- 健康新聞標題的情緒傾向
- 財經標題正負向與後續市場指標的粗略關聯
- 社會新聞標題情緒與點擊互動傾向

### 適合方式
- 文本特徵擷取
- 情緒分數
- 分組比較
- 簡單預測

### 風險
- 很容易過度宣稱因果
- 若做預測，必須小心資料切分與洩漏

## 研究設計最小可行版本（MVP）

如果你只有普通高中生程度、3-6 週時間，建議照這個最小版本：

1. 選 1 個平台或 1 個資料集
2. 定義 2-3 類標籤
3. 蒐集 / 整理 800-2000 筆文本
4. 完成標註規則
5. 做 1 個 baseline
6. 做 1 個較強模型或 1 個較完整規則法
7. 回報 Precision / Recall / F1
8. 列 10 筆誤判案例
9. 畫 3-5 張圖
10. 寫清楚限制與改進方向

這樣就已經比大量空泛議題型小論文強很多。

## 題目修正範例

### 太空泛
- AI 假訊息對高中生之影響

### 修成可做
- 以繁體中文健康新聞文本建立假訊息分類模型之研究
- 不同提示語設計對詐騙簡訊辨識正確率之影響
- 校園匿名留言中高風險霸凌語氣的自動辨識

### 太社會科
- 探討高中生使用 Instagram 現況

### 修成資訊題
- Threads 公開貼文之互動量與文本特徵分析
- Instagram 留言文本之情緒分布與主題比較
- 社群平台貼文字數、時段與互動量之關聯分析

## 寫作時一定要交代的 12 件事

1. 研究問題
2. 資料來源
3. 蒐集日期
4. 保留筆數
5. 分析單位
6. 標註規則
7. 前處理方式
8. baseline
9. 較強方法
10. 評估指標
11. 錯誤分析
12. 限制

## 常見扣分點

- 只說用了 AI，沒說用了什麼模型
- 只說準確率很高，沒說資料怎麼分
- 沒有 baseline
- 沒有樣本數
- 沒有標註原則
- 把公開留言直接當成全體民意
- 用很大的社會結論包裝很小的資料
- 沒有錯誤分析

## 如果題目本來是問卷型，怎麼救

原本：
- 探討高中生對 AI 假訊息的看法

救法：
- 主體改成公開文本分類
- 問卷退到補充角色，只用來討論使用經驗
- 主證據改由文字資料與模型評估提供

## 可以直接模仿的結構

1. 前言：問題背景 + 為什麼公開文字值得分析
2. 文獻探討：相關研究、文字分析方法、資料來源特性
3. 研究方法：資料蒐集、清理、標註、模型、指標
4. 結果與分析：各模型表現、圖表、誤判案例
5. 結論與建議：研究發現、限制、未來可擴充方向

## 最後判斷規則

如果一個題目同時符合以下四點，就值得優先做：
- 拿得到公開文字資料
- 可以定義清楚標籤或分析單位
- 能做 baseline 與量化結果
- 能產出圖表與誤判案例

如果只符合這兩點，通常要再修：
- 題目很熱門
- 大家有感

熱門不等於好做；能驗證才是好題目。
