# Guardian UI lessons

The operator should be able to answer within seconds:
- what broke?
- what evidence was seen?
- what diagnosis produced the current action?
- what command just ran?
- did it work?

## Minimum panels

- top health strip
- diagnosis card
- last action/result card
- recent event timeline
- safe manual controls
- link to latest reports/logs

## Avoid

- generic pretty dashboards with no operational meaning
- showing actions without showing evidence or diagnosis first
