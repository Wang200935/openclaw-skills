# Data Dictionary

適用專題：中文新聞標題四分類研究（政治、社會、國際、生活）

這份文件是第一版資料字典，目的不是把欄位做得很花，而是先把資料集的核心欄位定義清楚，讓後續蒐集、清理、標註、建模、誤差分析都能用同一套規格。

## 使用原則

- 研究主體是「新聞標題分類」，因此欄位設計以標題、標籤、來源、日期、可追溯性為主。
- `category_raw` 要保留，不能只留最後標籤，否則無法說明映射過程。
- `category_final` 固定為 4 類：`政治`、`社會`、`國際`、`生活`。
- 模糊案例要盡量寫入 `note`，方便人工複核與論文方法說明。
- 若某筆資料不納入正式訓練集，也建議保留紀錄並用 `is_included` 標示。

## First-pass data dictionary

| Field | Type | Description | Allowed values / format | Example value | Why it matters |
|---|---|---|---|---|---|
| `id` | string | 每筆新聞資料的唯一識別碼。 | 建議格式：`source_YYYYMMDD_流水號` 或穩定雜湊值；不可重複。 | `cna_20260310_001` | 方便去重、追蹤資料列、連結清理結果與模型錯誤案例。 |
| `title` | string | 新聞標題原文。 | 保留原始文字；不人工改寫；不可留空。 | `立院三讀通過新法案` | 這是分類模型的主要輸入，也是研究分析的核心文本。 |
| `category_raw` | string | 新聞網站原始提供的分類名稱。 | 依來源網站實際欄位保存，例如：`政治`、`社會`、`國際`、`生活`、`財經`、`兩岸`、`交通`、`娛樂`。 | `交通` | 可向評審展示「原始分類很多且不一致」，也是後續映射與標註修正的依據。 |
| `category_final` | string | 依研究規則統一後的最終標籤。 | 僅允許：`政治`、`社會`、`國際`、`生活`。 | `生活` | 這是模型訓練與評估的標準答案欄位，必須固定且一致。 |
| `source` | string | 新聞來源媒體名稱。 | 建議使用固定名稱，不混用簡稱與全名；例如：`中央社`、`自由時報`、`ETtoday`。 | `中央社` | 可分析不同媒體分布，也能避免模型其實只學到單一媒體風格卻沒被發現。 |
| `date` | date / string | 新聞發布日期。 | 建議統一格式：`YYYY-MM-DD`。 | `2026-03-10` | 可控制研究時間範圍，做時間切分、資料描述統計與重現蒐集期間。 |
| `url` | string | 新聞原始連結。 | 應為可追溯的完整網址；不可只存相對路徑。 | `https://example.com/news/a1` | 可回查原文、驗證標題、做去重，並提高研究資料透明度。 |
| `collected_at` | datetime / string | 實際蒐集或寫入資料集的時間。 | 建議 ISO 8601，例如：`YYYY-MM-DDTHH:MM:SS+08:00`。 | `2026-03-10T14:20:00+08:00` | 能區分「新聞發布時間」與「資料蒐集時間」，有助於重現流程與排查爬蟲問題。 |
| `note` | string, optional | 人工備註欄，記錄模糊案例、修正原因或特殊情況。 | 可空白；若有改標、排除、邊界案例，建議填寫簡短文字。 | `原始分類為財經，但主軸是央行政策，改標政治` | 是方法透明度的重要補強欄位，尤其能支撐邊界案例討論與誤差分析。 |
| `is_included` | boolean, optional | 這筆資料是否納入正式研究資料集。 | `true` / `false` | `true` | 可以保留被排除資料的痕跡，避免清理過程不可追蹤，也方便報告排除規則。 |
| `mapping_rule` | string, optional | 記錄這筆資料從原始分類映射到最終標籤時使用的規則。 | 建議短句描述，例如：`交通→生活（日常民生資訊）`、`兩岸→國際（區域局勢）`。 | `交通→生活（日常民生資訊）` | 能把人工判斷標準具體化，讓資料標註規則更容易重現與說明。 |

## Final label definition (`category_final`)

| Final label | Meaning | Typical title focus |
|---|---|---|
| `政治` | 政府、政黨、選舉、立法院、政策攻防、施政決策 | 選舉、法案、部會、內閣、政黨表態 |
| `社會` | 治安、犯罪、事故、司法、公共事件、制度衝突 | 車禍、火警、詐騙、起訴、校園事件 |
| `國際` | 外國政局、外交、戰爭、跨國事件、全球局勢 | 美國、日本、俄烏、聯合國、峰會 |
| `生活` | 日常民生、消費、旅遊、美食、交通資訊、保健、娛樂 | 優惠、旅遊、天氣、通勤、影劇、保健 |

## Common raw-label handling examples

| Raw label example | Usually map to | Notes |
|---|---|---|
| `政治` / `選舉` / `政府` / `國會` | `政治` | 直接屬於政治核心主題。 |
| `社會` / `治安` / `司法` / `事故` | `社會` | 與公共事件、犯罪、事故、司法有關。 |
| `國際` / `外交` / `兩岸` / `戰爭` | `國際` | 若主軸是國與國關係或全球局勢，優先歸國際。 |
| `生活` / `消費` / `旅遊` / `美食` / `娛樂` / `交通` | `生活` | 日常資訊、軟性民生內容通常歸生活。 |
| `財經` | 視主軸改歸，否則排除 | 例如央行政策可歸政治，國際油價可歸國際。 |
| `科技` / `體育` / `評論` / `未分類` | 多半排除或個案判斷 | 若無法穩定映射到四分類，建議不納入正式資料集。 |

## Recommended validation rules

1. `id`, `title`, `category_raw`, `category_final`, `source`, `date`, `url`, `collected_at` 為核心欄位。
2. `category_final` 不可出現四分類以外的值。
3. `title` 不可為空字串，也不應只剩標點符號。
4. `url` 應盡量唯一；若重複，需檢查是否為重複新聞。
5. `date` 與 `collected_at` 要分開保存，不能混成同一欄。
6. 只要有人工改標、模糊判斷、排除理由，至少填 `note` 或 `mapping_rule` 其中一欄。
7. 若 `is_included = false`，建議在 `note` 中寫明排除原因，例如 `評論文`、`無法判定主題`、`不屬四分類`。

## Example row

```csv
id,title,category_raw,category_final,source,date,url,collected_at,note,is_included,mapping_rule
cna_20260310_001,連假國道疏運懶人包出爐,交通,生活,中央社,2026-03-10,https://example.com/news/a1,2026-03-10T14:20:00+08:00,原始分類為交通，依日常民生資訊改標生活,true,交通→生活（日常民生資訊）
```

## Why this dictionary helps the paper

這份資料字典能支撐論文中幾個很重要的說明：

- 資料不是隨便抓下來就用，而是有欄位設計與研究規則。
- 原始分類與研究分類不同，因此需要標準化映射。
- 每筆資料都保留來源、日期、網址與備註，能提高可追溯性。
- 後續做資料清理、模型訓練、錯誤分析、圖表整理時，欄位定義不會一直變動。

如果後面開始正式蒐集資料，這份第一版資料字典還可以再升級成：
- CSV schema 檢查規則
- 標註手冊（annotation guideline）
- 清理流程 SOP
- 錯誤分析欄位補充表
