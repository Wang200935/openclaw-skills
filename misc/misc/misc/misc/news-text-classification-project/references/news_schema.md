# News Title Classification Schema

適用主題：中文新聞標題四分類研究（政治、社會、國際、生活）

## 1. 研究標籤定義

本研究固定使用 4 個最終類別（`category_final`）：

1. `政治`
   - 與政府、政黨、選舉、立法院、政策攻防、施政決策直接相關。
   - 關鍵詞例：選舉、立院、部會、政黨、政策、總統、行政院。

2. `社會`
   - 與治安、犯罪、事故、司法、公共事件、社會爭議、制度衝突直接相關。
   - 關鍵詞例：車禍、火警、詐騙、起訴、判決、勞資、校園事件。

3. `國際`
   - 與外國政局、戰爭、外交、國際組織、跨國事件、全球重大局勢直接相關。
   - 關鍵詞例：美國、日本、中國、俄烏、聯合國、峰會、制裁。

4. `生活`
   - 與日常生活、消費、美食、旅遊、交通資訊、保健知識、娛樂與軟性民生內容相關。
   - 關鍵詞例：優惠、旅遊、天氣、美食、通勤、保健、影劇。

## 2. 原始類別到最終類別的映射原則

資料來源網站常有很多原始分類（`category_raw`），例如：財經、娛樂、地方、生活、國際、兩岸、社會、政治、教育、健康、科技、體育。

為了讓研究問題更明確、類別更穩定，本研究將各站的原始分類統一映射到四個最終類別。

### 2.1 映射步驟

1. **先看標題主軸，不只看網站原始分類**。
   - 同一網站的分類標準不一定一致，不能完全照單全收。
2. **優先判斷「這則標題最核心在講什麼」**。
   - 例如「立院三讀通過」即使影響民生，主軸仍是政治。
3. **若原始類別不在四分類內，依研究規則改歸或排除**。
   - 例如財經、科技、體育並非本研究主類別。
4. **遇到模糊案例，記錄在 `note` 欄位**。
   - 方便後續人工複核與論文方法說明。

### 2.2 優先判斷規則

- **政治優先**：凡主軸是選舉、政黨、政府決策、議會攻防，優先歸 `政治`。
- **國際優先**：凡主軸是外國、戰爭、外交、國際局勢，優先歸 `國際`。
- **社會優先**：凡主軸是事故、犯罪、司法、社會衝突、公共事件，優先歸 `社會`。
- **生活兜底**：偏向日常資訊、消費、旅遊、美食、健康提醒、娛樂內容時歸 `生活`。

### 2.3 邊界案例處理

- `外交`：若重點是國與國互動，歸 `國際`。
- `兩岸`：若重點是台海或中外關係，歸 `國際`。
- `地方`：地方公共事件多歸 `社會`；地方選舉或施政攻防可歸 `政治`。
- `教育`：校園事件、制度衝突多歸 `社會`；純升學攻略不建議納入。
- `健康/保健`：制度與公衛政策歸 `社會`；日常保健知識歸 `生活`。
- `氣象`：日常天氣提醒歸 `生活`；重大災害現場事件歸 `社會`。
- `財經/股市/科技/體育`：本研究不單列。若能依主軸合理改歸四類則改歸，否則排除。
- `評論/社論`：非事件導向標題原則上排除，以避免主觀性過高。

> 實作時可配合 `label_mapping.csv` 做半自動映射，但最後仍需人工抽查。

## 3. 資料欄位定義

以下欄位適合作為資料集標準 schema。

### 必要欄位

- `id`
  - 型別：string
  - 說明：每筆新聞唯一識別碼。
  - 建議格式：`source_YYYYMMDD_流水號` 或雜湊值。

- `title`
  - 型別：string
  - 說明：新聞標題原文。
  - 規則：保留原始文字，不先做人工改寫。

- `category_raw`
  - 型別：string
  - 說明：原始網站提供的分類名稱。
  - 範例：`政治`、`社會`、`國際`、`生活`、`財經`、`兩岸`。

- `category_final`
  - 型別：string
  - 說明：依本研究規則轉換後的最終標籤。
  - 允許值：`政治`、`社會`、`國際`、`生活`。

- `source`
  - 型別：string
  - 說明：新聞來源媒體或網站名稱。
  - 範例：`中央社`、`自由時報`、`ETtoday`。

- `date`
  - 型別：string 或 date
  - 說明：新聞發布日期。
  - 建議格式：`YYYY-MM-DD`。

- `url`
  - 型別：string
  - 說明：新聞原始連結。
  - 規則：盡量保留可追溯來源。

- `collected_at`
  - 型別：string or datetime
  - 說明：實際爬取或整理資料的時間。
  - 建議格式：ISO 8601。

### 建議欄位

- `note`
  - 型別：string
  - 說明：人工備註，記錄模糊案例、排除原因或重新標記理由。
  - 範例：`原始類別為財經，但標題主軸為央行政策，改標政治。`

- `is_included`
  - 型別：boolean
  - 說明：是否納入正式研究資料集。
  - 用途：讓排除資料也能保留追蹤紀錄。

- `mapping_rule`
  - 型別：string
  - 說明：本筆資料使用哪一條映射規則轉換。
  - 範例：`財經→政治（政策主軸）`。

## 4. 標註一致性建議

為了讓後續模型訓練更可靠，建議採用以下規則：

1. 先建立 30–50 筆小樣本做人工試標。
2. 把容易混淆的標題整理成邊界案例表。
3. 同一批資料盡量由同一套規則標註。
4. 若多人標註，先做交叉檢查，再統一修訂規則。
5. 論文中可說明：本研究以「標題主軸分類」作為核心標註原則。

## 5. 一筆資料示例

```csv
id,title,category_raw,category_final,source,date,url,collected_at,note
cna_20260310_001,立院三讀通過新法案,政治,政治,中央社,2026-03-10,https://example.com/a,2026-03-10T14:20:00+08:00,
ettoday_20260310_002,連假國道疏運懶人包出爐,交通,生活,ETtoday,2026-03-10,https://example.com/b,2026-03-10T14:21:00+08:00,原始分類為交通，依日常民生資訊改標生活
```

## 6. 給主研究流程的實務建議

- 收資料時保留 `category_raw`，不要只留最終標籤。
- 建模前先做一次映射與人工抽查，避免錯誤標籤直接進模型。
- 最終論文可展示：
  1. 原始分類很多且不一致；
  2. 研究者設計四分類架構；
  3. 透過明確規則做標準化；
  4. 再進行文字分類實驗。

這樣會讓整個研究看起來更完整，也更像一個有方法設計的資訊專題。
