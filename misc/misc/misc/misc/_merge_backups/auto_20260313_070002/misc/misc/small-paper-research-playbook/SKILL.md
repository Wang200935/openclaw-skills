# Small Paper Research Playbook

Turn a broad idea into a research plan that can survive writing and judging.

## Core role

This skill is for planning, screening, and method design.

Use it to prevent weak topic choices early. Do not use it as the main writing skill once the topic, data, method, and evaluation plan are already settled.

## Core workflow

1. Restate the user's idea as a concrete research problem.
2. Check whether the topic has a workable evidence chain:
   - clear question
   - obtainable data
   - usable method
   - comparison or baseline
   - measurable result
   - realistic limitation
3. Reject or repair topics that are too broad, too trend-driven, or too dependent on questionnaire-only evidence.
4. Classify the project into one main route:
   - public-text / NLP
   - questionnaire-supported topic
   - system / prototype / app
   - analysis / prediction / comparison
5. Design evaluation before polishing the title.
6. Once topic, data, method, and evaluation are stable, hand off to `small-paper-writer` for drafting.

## Fast screening rules

Prefer topics with:
- real or public data
- reproducible collection steps
- explicit comparison, baseline, or before/after design
- measurable outputs
- a scope small enough for a student team

Be cautious with topics that have:
- hot buzzwords but vague execution
- only questionnaire charts and broad claims
- no clear dataset, no baseline, or no evaluation plan
- conclusions much bigger than the available evidence

## Output target

When helping the user plan a paper, try to leave them with a compact research design containing:
- provisional title
- research question
- data source
- method
- comparison / baseline
- evaluation metric
- likely limitation
- next step

## Handoff rule

Switch to `small-paper-writer` once the user already has:
- a bounded topic
- a usable data source
- a method choice
- an evaluation plan

At that point, stop re-planning and start drafting.
