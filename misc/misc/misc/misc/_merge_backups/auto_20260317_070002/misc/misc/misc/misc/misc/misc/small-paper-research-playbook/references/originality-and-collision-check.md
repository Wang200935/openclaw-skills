# 原創性與撞題檢查指南

這份文件是給已經有題目方向、但還沒正式定題的人用的。

目的不是把題目包裝得「看起來不像」，而是幫你誠實判斷：
- 這題是不是其實已經很多人做過
- 你和既有作品到底差在哪裡
- 哪些地方可以借骨架，哪些地方不能直接沿用
- 怎麼寫出站得住腳的原創性說明

依 404 篇資訊類小論文 corpus 來看，**真正的撞題，常常不是標題一字不差，而是研究問題、資料、方法、驗證幾乎同一套。**

---

## 一、先有一個基本觀念：撞題看的是「研究骨架」，不是只看標題

兩篇題目就算名字不同，也可能本質上幾乎同題。

### 例子：表面不同，骨架其實很像
- 「短影音的發展—探討短影音迅速竄紅背後的原因與影響」
- 「短影音的影響與喜好度」
- 「高中生喜愛短影音的類別及短影音崛起的因素探討」
- 「短影音之影響與高中生使用習慣研究探討」
- 「短影音對社會文化的塑造與挑戰：一個新媒體現象的研究」
- 「短影音的崛起：興起與趨勢分析」

這些不一定互相抄襲，但在評審視角裡，會被歸到同一個過熱簇：
**短影音 + 高中生 / 社會影響 + 問卷 / 文獻整理**。

### 例子：同一熱區反覆出現
- ChatGPT / AI 對學習影響
- AI 繪圖優劣與接受度
- 假訊息 / AI 假訊息影響
- Threads 與 Instagram 比較
- VTuber 發展 / 經營 / 觀眾偏好
- YOLO 影像辨識應用
- Unity / RPG / 2D 遊戲製作
- RAG / LLM 校園助理或記事應用

所以判斷撞題時，不能只問：
- 標題有沒有一樣？

而要問：
- 研究問題是否同一類？
- 資料來源是否同一類？
- 方法是否同一類？
- 驗證是否同一類？
- 做完後會不會像是「上一屆作品換一個名詞」？

---

## 二、用 corpus 判斷撞題的四層檢查法

可直接用下面四層來檢查。

## Layer 1：主題層
先看大題目是不是已經高度擁擠。

### 高擁擠題群（corpus 中反覆出現）
- 短影音影響 / 使用習慣
- ChatGPT / 生成式 AI 對學習或生活的影響
- AI 繪圖的優缺點 / 接觸情形 / 應用
- 假訊息 / AI 假訊息 / 媒體識讀
- Threads / Instagram / 社群平台偏好
- VTuber 發展、經營、比較
- Unity / RPG / 2D 遊戲製作
- YOLO / 影像辨識套到某個應用
- 區塊鏈 / 量子 / 元宇宙的未來與影響

### 這一層怎麼判斷
如果你的題目一說出口，就能在 corpus 裡想到 5 篇以上同主題，先不要急著寫。
這不代表不能做，而是代表：
**你接下來必須證明自己不是同一簇的普通版本。**

## Layer 2：研究問題層
同主題不一定撞題；同研究問題就很容易撞。

### 例子
下面這些就很接近：
- 探討高中生對 ChatGPT 的使用現況
- ChatGPT 對高中職生學習的影響
- 高中生運用 ChatGPT 於課業學習的探究
- ChatGPT 對青少年學習及生活的影響

它們的核心研究問題都很像：
**某群學生怎麼用 ChatGPT，以及覺得它有什麼影響。**

如果你的問題句可以套進既有題目的模板，只改一個名詞，撞題風險就高。

## Layer 3：證據層
真正拉開差距的是你拿什麼證據回答問題。

### 高撞題證據組合
- 大主題 + Google 表單問卷
- 新技術名詞 + 文獻整理
- 系統原型 + 截圖展示 + 少量心得

### 較不容易撞題的證據組合
- 公開文本資料 + 明確標註規則 + baseline 比較
- 開放資料 + 可重現分析流程 + 誤差討論
- 系統原型 + 使用者任務測試 + 量化指標
- 同主題但換成不同資料型態與不同驗證問題

## Layer 4：評審感受層
最後問一題最實際：

**如果把你的題目和 corpus 裡最像的 2–3 篇排在一起，評審會覺得：**
- 這是同題換皮
- 還是這題真的有新的切法

如果直覺偏第一種，就還不夠。

---

## 三、快速撞題評分表

可以用這張表先做粗判。

### 每項 0–2 分
- **主題重複度**
  - 0：corpus 中很少見
  - 1：有幾篇相近
  - 2：很多篇都在做
- **研究問題相似度**
  - 0：問題問法明顯不同
  - 1：部分重疊
  - 2：核心問題幾乎一樣
- **資料來源相似度**
  - 0：資料型態明顯不同
  - 1：部分相似
  - 2：同平台 / 同資料型態 / 同樣本群
- **方法相似度**
  - 0：方法架構不同
  - 1：有共通元素
  - 2：幾乎同一流程
- **驗證方式相似度**
  - 0：指標 / 比較設計不同
  - 1：部分相似
  - 2：幾乎同一套驗證

### 解讀
- **0–3 分**：低撞題風險
- **4–6 分**：中度風險，要明確做差異化
- **7–10 分**：高風險，通常應改題或大修設計

## 四、怎麼做「誠實差異化」

差異化不是把標題換句話說，而是讓研究真的多出新的資訊價值。

## 安全做法 1：換研究問題，不只換主題名詞

### 不夠
- 把「ChatGPT」換成「Gemini」
- 把「Instagram」換成「Threads」
- 把「短影音」換成「Reels」

### 比較安全
- 從「影響」改成「錯誤類型比較」
- 從「使用意願」改成「不同提示語對正確率的影響」
- 從「平台偏好」改成「公開貼文互動特徵與文本特徵分析」
- 從「AI 很厲害嗎」改成「哪一類任務表現失敗最多」

## 安全做法 2：換證據，不只換對象

### 不夠
- 去年做高中生，今年改國中生
- 去年做某校，今年改另一校

如果方法還是同一份問卷、同一種文獻整理，差異通常很弱。

### 比較安全
- 問卷題改成公開資料題
- 心得題改成任務測試題
- 單一模型展示改成 baseline 比較
- 純平台介紹改成可量化資料分析

## 安全做法 3：換驗證邏輯

### 例子
YOLO / 影像辨識題很多，如果只是：
- 做一個辨識系統
- 展示可以抓到東西

那很容易跟大量既有題撞骨架。

但如果你改成：
- 比較不同模型在同一資料集上的 precision / recall / latency
- 比較不同拍攝角度、遮擋程度下的失誤
- 討論資料偏差與誤判場景

就從「demo 題」變成比較像研究題。

## 安全做法 4：縮到自己的真實場景

強而安全的差異化常來自：
- 有自己取得的真實資料
- 有自己熟悉的在地情境
- 有自己能做的測試流程

例如：
- 校園匿名留言情緒與霸凌預警
- 某類繁中評論的語言特徵
- 某校園網站資訊檢索困難的 RAG / 搜尋比較
- 某類使用者任務的系統表現差異

這種比泛泛談「AI 對社會影響」安全很多。

## 五、怎麼借研究骨架，而不變成抄題

這是最重要的一段。

可以借的，是**研究 skeleton**；不能直接搬的，是**題目本身與證據本身**。

## 可以借的東西
- 問題收斂方式
- 方法章節順序
- baseline 設計概念
- 指標組合
- 結果圖表類型
- 誤差 / 限制寫法
- 系統驗證的任務設計框架

## 不可以直接借的東西
- 幾乎相同的題目句型與研究問題
- 同一資料來源但不做新定義
- 同一標註方式但不說明理由
- 同一系統流程只換介面或主角
- 同一章節內容與論述句

## 一個安全的借骨架流程

### Step 1：先找 1–3 篇最像的作品
記下每篇的：
- 研究問題
- 資料
- 方法
- baseline
- 指標
- 限制

### Step 2：只抽象化，不照抄內容
例如你從一篇文字分類研究抽出的是：
- 明確標註規則
- train/test 切分
- baseline 比較
- confusion matrix
- 錯誤分析

這叫借骨架。

如果你抽出的是：
- 題目句型
- 同一資料集
- 同一分類標籤
- 同一研究結論

這就太像。

### Step 3：至少做出 2 個實質差異
通常至少要在下列中做出兩項明顯改變：
- 研究問題
- 資料來源
- 標籤定義
- 方法設計
- 比較組
- 驗證指標
- 應用場景

### Step 4：最後再修標題
先確定研究真的不同，再寫標題。
不要先寫一個很新潮的標題，再拿舊骨架硬套。

## 六、哪些「差異」其實很假

下面這些常被誤以為是創新，但通常不夠。

## 1. 只換平台名
- Facebook 改 Instagram
- ChatGPT 改 Gemini
- TikTok 改 Reels

如果研究問題和方法不變，這通常只是平移。

## 2. 只換研究對象
- 高中生改大學生
- 某校改某校
- 某縣市改某縣市

若沒有理由說明這個對象改變會帶來理論或資料差異，創新度有限。

## 3. 只換時間
- 2024 改 2025
- 最新版模型改新版模型

除非你的核心研究就是版本變遷比較，不然這通常不算真正創新。

## 4. 只多一個工具名詞
- 原本是分類，現在加上「AI」
- 原本是搜尋，現在加上「RAG」
- 原本是聊天機器人，現在加上「多代理」

如果沒有 benchmark、任務集、比較與限制，評審很容易看出只是包裝升級。

## 七、corpus 給你的幾個高風險提醒

## 1. 社群 / 短影音 / 使用影響題很容易過熱
短影音、Instagram、Threads、社群媒體影響，404 篇裡反覆出現。

### 高風險版本
- 探討某平台對青少年的影響
- 探討某平台使用習慣
- 問卷調查偏好與看法

### 較安全版本
- 公開貼文互動特徵分析
- 留言文本情緒 / 主題分類
- 特定事件或特定社群的內容比較

## 2. ChatGPT / 生成式 AI 題很容易做成「時事心得」
corpus 中大量出現：
- 使用現況
- 學習影響
- 生活影響
- 認知與意願

### 高風險版本
- 高中生怎麼用 ChatGPT
- ChatGPT 對學習有什麼幫助

### 較安全版本
- 不同提示策略對特定任務表現的影響
- 不同模型在明確任務上的錯誤類型比較
- LLM + RAG 在單一資料域的回答品質比較

## 3. 系統原型題容易撞在「做出來了」
像：
- 智慧點名
- 校園助理
- 門禁辨識
- 機器人 / 物聯網裝置

### 危險點
很多題目都能做出功能，但如果沒有：
- 成效比較
- 場景測試
- 失敗分析

就很容易和既有作品撞在同一個展示層級。

## 4. AI / YOLO / 影像辨識題容易撞在「模型套場景」
如果題目只是：
- 用 YOLO 辨識某東西
- 用深度學習做某個偵測

這在 corpus 裡已非常多。

### 安全提升方式
- 換成「比較與評估」題
- 明確討論資料限制與失敗樣態
- 把問題縮到特定條件下的準確率差異

## 八、正式定題前一定要做的「三篇避撞表」

請至少找 3 篇最接近的既有作品，做成下面表格。

| 項目 | 你的題目 | 範例 A | 範例 B | 範例 C |
|---|---|---|---|---|
| 研究問題 |  |  |  |  |
| 資料來源 |  |  |  |  |
| 方法 |  |  |  |  |
| baseline / 比較 |  |  |  |  |
| 指標 |  |  |  |  |
| 主要新增價值 |  |  |  |  |

做完後再問：
- 我新增的是不是「真正的新資訊」？
- 還是只是「新的包裝」？

如果最後「主要新增價值」那格寫不出具體句子，通常還沒準備好定題。

## 九、原創性說明怎麼寫才誠實

原創性說明不是說「本研究前所未見」，而是清楚交代：
- 你知道前人做過什麼
- 你和前人哪裡相同
- 你和前人哪裡不同
- 你的新增價值是什麼
- 你不把自己的創新說過頭

## 好的原創性說明長什麼樣

### 結構模板
1. **先承認相近領域已有研究**
2. **指出既有研究常見做法或限制**
3. **說明你這題的差異點**
4. **說明差異帶來的新增價值**
5. **避免誇大成突破性發明**

### 範例句型 1：公開文字 / NLP 題
既有研究中，已有多篇作品探討假訊息、校園留言或社群文本的影響與辨識問題，但多數作品偏向問卷調查、整體影響討論，或僅展示模型功能。本研究延伸相同議題，改以特定繁體中文文本語料作為分析對象，明確建立標註規則，並加入 baseline 比較與錯誤分析，嘗試回答「哪些語言特徵最容易造成誤判」此更具體的研究問題。因此，本研究的原創性不在於首次討論該議題，而在於以不同證據設計與更可重現的評估流程，補足既有作品在驗證層面的不足。

### 範例句型 2：系統原型題
歷屆作品中已有智慧點名、校園助理與辨識系統等相關應用，顯示校園資訊場景是常見研究方向。本研究並非主張題材全新，而是聚焦於 ______ 的特定使用情境，與既有作品相比，在資料來源、任務流程與成效驗證方式上有所不同，特別加入 ______ 與 ______ 的比較，以檢驗系統在真實場景中的可用性。因此，本研究的價值在於針對既有題型做更聚焦且更具驗證性的延伸，而非僅以不同名稱重做同類系統。

### 範例句型 3：LLM / AI 工具比較題
近年已有多篇作品討論 ChatGPT、Gemini 或其他生成式 AI 的應用與影響，但不少研究停留於使用看法、整體印象或功能介紹。本研究改以 ______ 任務作為固定評測場景，比較不同提示策略 / 不同模型在相同條件下的表現差異，並分析其常見錯誤類型。故本研究的原創性主要體現在研究設計與評估框架，而非單純更換 AI 工具名稱。

## 十、寫原創性說明時不要犯的錯

## 1. 不要硬說「國內外尚無研究」
只要 corpus 裡已經一堆相近題，這句很危險。

比較安全的說法是：
- 既有作品已有相關探討，但在某一面向仍不足
- 目前常見做法多聚焦於 X，本研究改聚焦於 Y
- 現有研究多採用 A，本研究改採 B 進行比較

## 2. 不要把換平台名講成創新
- 把 ChatGPT 換成 DeepSeek
- 把 Instagram 換成 Threads

除非你研究的是平台差異本身，否則這不夠。

## 3. 不要把「有做出系統」就當原創
很多作品都能做出來。
真正有說服力的是：
- 為什麼這樣做
- 跟其他做法比怎樣
- 哪裡會失敗

## 4. 不要把借骨架說成完全獨創
誠實比較好。
評審通常不怕你參考前人，怕的是你明明站在前人肩膀上，卻假裝自己從零開始。

---

## 十一、最後的避撞題五問

正式定題前，至少回答這五題：

1. **如果拿 corpus 搜相近題，我最像哪三篇？**
2. **我和它們相比，至少哪兩項是實質不同？**
3. **我的新增價值是新資料、新問題、新方法，還是新驗證？**
4. **如果把標題遮住，光看方法與結果，會不會還是像舊題換皮？**
5. **我的原創性說明能不能誠實承認前人，再清楚說出自己的差異？**

只要這五題有兩題答不穩，就先不要急著定。

---

## 十二、最終原則

**避撞題不是避開所有相似，而是避開「沒有新增價值的相似」。**

真正安全的做法不是：
- 把標題改得更花
- 換幾個流行名詞
- 假裝沒看過前人

而是：
- 知道哪些題已經很多人做
- 承認自己借了哪些研究骨架
- 做出真正不同的資料、問題、方法或驗證
- 用誠實而具體的文字說明新增價值

這樣就算你做的是熱門題，也比較不會變成撞題版。
