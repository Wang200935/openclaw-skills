# 文獻探討到研究方法的橋接說明

## 這份橋接筆記的用途
這份文件的目的，是把第二章文獻探討與第三章研究方法接起來，避免論文出現「前面談理論，後面卻像突然換題目」的斷裂感。寫法上要讓評審看見：本研究的方法不是隨便挑的，而是根據前人對中文文本分類、短文本處理、特徵表示、基線模型與評估方式的共通知識一步一步設計出來。

---

## 一、從文獻探討導到研究方法的主線
文獻探討建議不要只是分段介紹名詞，而要形成下面這條邏輯鏈：

1. **文字分類是一個成熟的自然語言處理任務**，新聞分類是其中常見應用。
2. **中文新聞標題屬於短文本**，字數少、上下文不足，因此特徵設計會特別重要。
3. **中文沒有天然空格**，所以斷詞與中文前處理會直接影響後續特徵品質。
4. **傳統文字特徵表示法**如 CountVectorizer 與 TF-IDF，仍是短文本分類中可解釋、可比較、適合教學研究的基礎方法。
5. **經典分類模型**如 Multinomial Naive Bayes、Logistic Regression、Linear SVM，常被用作文本分類基線與比較模型。
6. **若資料可能不平衡，不能只看 Accuracy**，還要看 Precision、Recall、Macro F1 與混淆矩陣。
7. **分數之外還需要錯誤分析**，才能說明模型為什麼會判錯，也才能回應中文短文本分類的真正挑戰。

這樣一來，方法章的每一個設計選項，都能回扣到文獻探討中的理論依據。

---

## 二、文獻探討各節要如何對應到方法章

### 1. 文字分類基本概念 → 為何研究新聞標題分類
文獻探討先說明文字分類的基本流程：資料蒐集、前處理、特徵表示、模型訓練、評估。接著指出新聞分類是典型應用，而新聞標題又屬於短文本分類。

**方法章的對應寫法：**
因此，本研究以中文新聞標題作為研究對象，依照文字分類的一般流程，建立資料集、完成前處理、進行特徵轉換與模型比較，並以標準評估指標分析其分類效果。

### 2. 中文文本處理與斷詞文獻 → 為何要設計斷詞與前處理
文獻探討若提到中文沒有空格、斷詞結果會影響語意單位保留、專有名詞切分不當會降低模型辨識能力，那就自然能導出本研究必須把「中文斷詞」當成方法變項之一。

**方法章的對應寫法：**
基於中文文本處理研究指出斷詞品質會影響特徵表示效果，本研究在前處理階段將設計中文斷詞策略，並比較不同切分方式對新聞標題分類結果的影響。

### 3. 特徵表示文獻 → 為何選 TF-IDF 為核心方法
文獻探討應指出：文字不能直接輸入模型，必須先轉成數值特徵；其中 CountVectorizer 代表基本詞頻，TF-IDF 則可降低高頻但區辨力不足的詞，常用於文本分類任務。

**方法章的對應寫法：**
因此，本研究將以 TF-IDF 作為核心特徵表示方法，並與基本詞頻表示作比較，觀察在中文新聞標題這種短文本情境下，哪種表示方式能保留較好的分類訊息。

### 4. 基線模型文獻 → 為何選 Naive Bayes、Logistic Regression、Linear SVM
文獻探討若整理出這三類模型都是文字分類常見基線，就能讓方法章中的模型選擇顯得有根據，而不是任意挑選。

**方法章的對應寫法：**
根據文本分類相關研究，Multinomial Naive Bayes、Logistic Regression 與 Linear SVM 都是處理高維稀疏文字特徵時常見且具代表性的基線模型，因此本研究選用這三者作為主要比較對象。

### 5. 評估文獻 → 為何不能只看 Accuracy
文獻探討要先說明類別不平衡時 Accuracy 的侷限，再引出 Precision、Recall、F1-score 與 Macro F1 的必要性，最後提到混淆矩陣能揭露類別間混淆關係。

**方法章的對應寫法：**
因此，本研究不僅以 Accuracy 作為整體參考，也將 Macro F1 列為核心指標，並搭配各類別 Precision、Recall 與混淆矩陣，以更完整呈現模型在不同新聞類別上的實際表現。

### 6. 文獻中的研究限制與短文本挑戰 → 為何一定要做錯誤分析
如果文獻探討指出短文本常因資訊不足、語詞重疊、專有名詞或類別邊界模糊而產生誤判，方法章就應安排錯誤分析，而不是只放結果表格。

**方法章的對應寫法：**
考量中文新聞標題具有篇幅短、語境不足與跨類別詞彙重疊等問題，本研究將在模型評估後進一步進行錯誤分析，整理常見誤判案例與可能原因，作為結果討論的重要依據。

## 三、這篇研究最關鍵的五個方法連結

### （一）文獻談中文處理特殊性，所以方法要做中文斷詞
這不是額外裝飾，而是直接回應中文文本的基本特性。若文獻強調中文沒有空格、專有名詞切分重要，那方法章就要明寫：本研究將中文斷詞視為前處理重點，並觀察斷詞對分類表現的影響。

### （二）文獻談短文本特徵稀疏，所以方法要用 TF-IDF
新聞標題很短，常只有少量高辨識詞。TF-IDF 的合理性就在於：它能讓真正有區辨力的詞更突出，降低過度常見詞的影響。因此文獻探討談完特徵表示之後，方法章就自然接到 TF-IDF。

### （三）文獻談經典文本分類流程，所以方法要先做 baseline
高中小論文不需要一開始追求黑箱或最先進模型。反而應該先用可解釋的基線模型建立比較架構。這就是為什麼方法章要從 Naive Bayes、Logistic Regression、Linear SVM 開始，而不是直接跳深度學習。

### （四）文獻談類別不平衡，所以方法要用 Macro F1
如果資料中某些類別樣本較多，只看 Accuracy 容易高估表現。文獻探討要先講這件事，方法章再明確說：本研究以 Macro F1 作為重要指標之一，避免模型只對大類別表現好卻掩蓋小類別失敗。

### （五）文獻談短文本容易混淆，所以方法要安排錯誤分析
真正有說服力的研究，不只比較分數，還要解釋錯在哪裡。尤其新聞標題可能出現政治／國際、社會／生活等邊界重疊情況，所以方法章必須規劃錯誤案例表與混淆矩陣分析。

## 四、可直接放進論文的橋接段落範例

### 範例 1：文獻探討結尾接研究方法
綜合前述文獻可知，文字分類已廣泛應用於新聞整理等任務，而中文新聞標題屬於短文本，具有字數少、語境不足與詞彙切分困難等特性。相關研究亦指出，中文斷詞、文字特徵表示方式與分類模型選擇，都會影響最終分類效果。此外，在類別可能不平衡的情況下，若僅以 Accuracy 評估，容易高估模型實際能力。因此，本研究將以中文新聞標題為對象，設計包含前處理、中文斷詞、TF-IDF 特徵轉換、基線模型比較、Macro F1 與混淆矩陣評估，以及錯誤分析的完整研究流程，以探討何種方法組合較適合本研究資料。

### 範例 2：方法章開頭說明
根據文獻探討，中文短文本分類的關鍵在於前處理品質、特徵表示方式與模型比較設計，因此本研究在方法上依序進行資料蒐集與清理、中文斷詞、特徵擷取、分類模型訓練、指標評估與錯誤分析。其中，特徵表示以 TF-IDF 為核心，並搭配文字分類常見的基線模型進行比較，以建立一套可重現且具解釋性的中文新聞標題分類研究流程。

## 五、這份橋接對本研究的實際作用
簡單說，第二章文獻探討不是為了「介紹很多名詞」，而是為了替第三章研究方法提供理由。這篇研究最理想的連接方式是：

- 因為文獻說中文處理很重要，所以方法要做斷詞。
- 因為文獻說文字要轉成可計算特徵，所以方法選 TF-IDF。
- 因為文獻說經典模型適合文本分類基線，所以方法比較 NB、LR、SVM。
- 因為文獻說資料可能不平衡，所以方法看 Macro F1，不只看 Accuracy。
- 因為文獻說短文本易混淆，所以方法一定要做混淆矩陣與錯誤分析。

如果照這個結構寫，評審通常會很容易看出：這篇論文的方法不是東拼西湊，而是從文獻探討自然推導出來的。
