# AI Evals / Observability

Use this skill to make AI behavior inspectable and testable.

## Core workflow

1. Define what success looks like.
2. Capture inputs, outputs, and important intermediate steps.
3. Save representative failures.
4. Turn recurring failures into regression tests.
5. Compare prompt or routing variants systematically.

## Good defaults

- Keep a corpus of small test cases.
- Store failure examples near the code or workflow.
- Prefer structured logs over vague prose.
- Separate model-quality issues from transport/runtime issues.
